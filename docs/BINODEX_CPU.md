# binodex: снижение CPU headless-рендера (фон `.wrap_bg` + off-zone)

Дата исследования: 2026-06-12. Probe-скрипты: `scripts/probe_binodex_*.py`
(каталог `scripts/` в `.gitignore` — скрипты дев-онструмент; знание здесь).

## ⭐ РЕШЕНИЕ (2026-06-14): глобус выключается настройкой аккаунта

Глобус (`.wrap_bg`) управляется настройкой binodex **«Тема»** (`#setup_theme_toggle`):
inactive → глобус ВЫКЛ (`.wrap_bg` удаляется из DOM), active → ВКЛ. Настройка персистит за
аккаунтом, и **у свежего аккаунта глобус ВЫКЛ ПО УМОЛЧАНИЮ** (проверено холодным логином,
`scripts/probe_default_globe.py`). Корень 90% CPU в проде: старый `_setup`
(`apps/binodex_session.py`) слепым кликом тумблера САМ включал глобус. **Исправлено**: `_setup`
жмёт тумблер только если `.wrap_bg` реально присутствует (детерминированно «ВЫКЛ»). ⇒ новые
куки → глобус ВЫКЛ → CPU ~90→~38% на стороне сайта, без PIL-композита и без риска для детекта
кук. Цена: посты без глобуса (чарт на сплошном тёмном фоне). Прод сейчас на старых куках
(глобус ВКЛ) — нужен перевыпуск кук (`do_setup=True`) или авто-рефреш. Пайплайн ниже —
опция, если глобус в кадре всё-таки нужен.

## Диагноз

CPU headless-Firefox на `binodex.app/trade` (зал. терминал, вьюпорт 1712×990,
замер по дереву процессов `ms-playwright/firefox` через `/proc`):

| Состояние | CPU |
|---|---|
| как есть | **~90–96 %** |
| `.wrap_bg` → `display:none` | **~38–39 %** (−56 пт) |
| `.wrap_bg` off + всё вне зоны скрина → `visibility:hidden` | **~21–22 %** (−69 пт) |

**Виновник №1** — `.wrap_bg`: полноэкранный `position:fixed; z-index:-1` слой с
`background: linear-gradient(rgba(20,23,31,0.5)…), url(<земля>)`. Полупрозрачный →
браузер переблендивает его поверх живого канваса каждый кадр (софт-композит без GPU).

**Виновник №2** — UI вне зоны скрина (правое торговое меню, аккаунт-бар, анимации):
рендерится зря, в кадр не попадает. Скрытие даёт ещё ~−17 пт.

Не помогает: блокировка рекламы/трекеров, заморозка CSS-анимаций (нагрузка render-bound,
не network). Подробные замеры — в памяти Claude `resource-profile-screens`.

## Что снимает прод

`screen_zone` (binodex_settings) = ОДИН канвас
`.graph > div:nth-child(2) > div:nth-child(1) > canvas:nth-child(2)` (~1470×870).
Канвас на **97% прозрачный** и держит ВСЁ содержательное: свечи, оси цены/времени,
часы «Время закрытия», ценник, линию экспирации. Глобус виден сквозь него (это фон
`.wrap_bg`). Вне канваса — только HTML: тулбар, лейбл пары, `H1` + правое меню.

## Пайплайн (Вариант Б — глобус постобработкой, CPU ~21 %)

Идея: не рендерить живой `.wrap_bg`, а возвращать глобус композитом в PIL.
`omit_background` в Firefox НЕ поддержан → альфу берём из канваса напрямую.

```
РАЗОВО (на пару):  глобус-ассет = клип бокса при скрытом канвасе и живом .wrap_bg,
                   с видимым лейблом пары (запекается в ассет).
СТЕАДИ:            .wrap_bg → display:none;
                   всё кроме канваса/.wrap_bg → visibility:hidden;
                   лейбл пары → visibility:visible (детект блока .graph_pair_wrap
                   по содержимому+геометрии — устойчиво к ротации классов).
КАДР:              canvas.toDataURL('image/png')  →  свечи с альфой;
                   Image.alpha_composite(globe, canvas);
                   paste_overlay(QR, otc_qr_x=1360, otc_qr_y=757).
```

Композит совпадает с текущим прод-кадром на **~0.4 %** (mean abs diff <1/255).
Матирование/numpy не нужны — альфа точная из канваса.

## ✅ Внедрено в прод (2026-06-14)

`apps/otc_app.py::screenshot_otc` собирает кадр композитом: **глобус(файл) + канвас(toDataURL)
+ ярлык пары(вырезка) + QR**. Глобус выключен за аккаунтом (в рантайме не рендерится),
подкладывается из `pictures/globe_otc.png` (`globe_otc_path`). Ярлык пары — HTML, в toDataURL
не входит → кладётся отдельным слоем: вырезка по разнице A−B (регион «ярлык виден» минус
«ярлык скрыт» при выкл глобусе → прозрачно без «короба»), кэш на пару. Helper'ы `_canvas_alpha`/
`_load_globe`/`_label_cutout`/`_matte_label`. Глобус в рантайме НЕ включаем (иначе теряется
экономия) — ярлык режем при выкл глобусе, глобус-файл печётся разово офлайн.

**off-zone ТОЖЕ внедрён (2026-06-14):** `_apply_offzone`/`_clear_offzone`/`_prepare_after_select`.
`visibility:hidden` на всё вне цепочки канваса, в белом списке — `#setup_settings_open` (детект кук
`_ui_loaded` в рантайме) и `#select_pair_add` (ярлык/кнопка модалки). Применяется в `init_otc` и в
конце успешного `select_otc_pair`; снимается в начале `select_otc_pair` (модалка выбора вне зоны).
Вырезку ярлыка строим, пока off-zone снят. Итог CPU **~18–22%** (≈4× от 90%); детект кук и выбор пары
целы (проверено прод-флоу).

## Внедрение в прод (исходный TODO — выполнено выше)

В `apps/otc_app.py::screenshot_otc`: заменить `element.screenshot()` на
toDataURL+композит; глобус-ассет снимать разово в `init_otc`/смене пары; стеади-инъекцию
CSS (`.wrap_bg` off + off-zone hidden) ставить один раз после загрузки терминала.
Проверить, что ~21 % держится в динамике (не только на статике).

⚠️ **НЕ прятать маркеры детекта отвала кук/сплеша.** Шаг «всё вне зоны → `visibility:hidden`»
прячет кнопку настроек `#setup_settings_open`, по ВИДИМОСТИ которой `_ui_loaded` судит «UI жив»
— а её зовёт не только `init_otc`, но и рантайм `otc_session_dead` (каждые `UI_DEAD_CONFIRM=3с`)
и `main_app.py` перед reload. Скрытие → `_ui_loaded`=False → ложный «сплеш/отвал кук» →
бот зря пересоздаёт браузер / релогинится. Поэтому `#setup_settings_open` держать в белом
списке `visibility:visible` (как лейбл пары) — он вне crop-зоны канваса, в кадр не попадёт, но
останется живым для детекта; CPU двух кнопок ≈ 0. Безопасны: `on_trade` (URL) и `feed_dead`
(WS) — DOM-независимы; `#select_pair_add` (readiness) = лейбл пары, и так оставляем видимым;
`#tek_val` в рантайме не используется. Альтернатива — off-zone CSS накладывать транзиентно
только внутри `screenshot_otc` и сразу снимать (детект работает между скринами).

## Для разработчиков binodex (если фон оставляют живым)

Проблема в методе подачи фона: полноэкранный полупрозрачный `linear-gradient`-оверлей
композитится софтом поверх анимированного канваса каждый кадр. Решение без изменения вида:
сделать фон непрозрачным (`opacity:1`, тон запечь в картинку) либо вынести `.wrap_bg`
на собственный GPU-слой (`will-change: transform` / `translateZ(0)`).
